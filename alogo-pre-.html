1. Introduction

Let me ask you something...

If I give you 1 million numbers, and ask you to find just ONE number… how will you search it FAST?

Linear Search

Check one by one → Worst case check all → O(n)

Binary Search

Needs sorted data → Divide into halves → O(log n)

But what if I tell you… we can find the element almost instantly?

That method is called Hashing.

2. Topics

  • Direct Address Tables

3. Direct Address Table

Index = Student ID

Value = Marks

Example: A[25] = 80

Time Complexity: O(1)

Problem: Wastes memory if IDs are large

4. Hash Tables

Use function instead of direct index

index = h(key)

Example: h(k) = k % 10

h(123) = 3 → Store at index 3

5. Collision

Example:

h(21) = 1

h(31) = 1

Collision occurs!

6. Collision Handling

Chaining

Index 1 → 21 → 31 → 41

Linear Probing

If full → move to next index

7. Problem Solving

Problem 1 (Chaining)

Insert: 23, 43, 13 with h(k)=k%10

23 % 10 = 3
43 % 10 = 3
13 % 10 = 3
Final: Index 3 → 23 → 43 → 13

Problem 2 (Linear Probing)

Insert: 10, 17, 24 with h(k)=k%7

10 % 7 = 3 → place at 3
17 % 7 = 3 → collision → 4
24 % 7 = 3 → collision → 5
Final: Index 3→10, 4→17, 5→24

Problem 3 (Search)

Find 22 in Index 2 → 12 → 22 → 32

Check list → Found 22
Time: O(1) average

Problem 4 (Load Factor)

n = 8, m = 10

λ = n/m = 8/10 = 0.8
High load → more collisions

8. Complexity

Search: O(1)

Insert: O(1)

Worst Case: O(n)

9. Conclusion

Hashing reduces search time from O(n) to O(1)

Used in:

  • Google Search
  • Databases
  • Password Systems
